Detalle técnico del módulo

Esta sección resume el enfoque metodológico, los tipos de SQLi abordados y la evidencia académica integrada en la adaptación web del proyecto.

Metodología aplicada
Burp Suite Web Security Academy

Los laboratorios fueron estructurados a partir de la identificación del punto vulnerable, la manipulación de peticiones HTTP, la construcción de payloads y la verificación de evidencia técnica mediante respuestas, mensajes, cambios de comportamiento o extracción de información.

  • Interceptación y modificación de requests.
  • Pruebas con payloads UNION, error-based, blind y out-of-band.
  • Validación del resultado con evidencia y estado “Lab Solved”.
  • Documentación resumida orientada a portafolio profesional.
Tipos de SQL Injection incluidos
Cobertura completa

El módulo integra distintos escenarios de explotación para mostrar la evolución técnica del tema, desde bypass de autenticación hasta exfiltración condicionada y evasión de filtros.

  • WHERE clause manipulation.
  • Login bypass.
  • UNION-based SQL Injection.
  • Error-based SQL Injection.
  • Blind SQL Injection: conditional responses / errors / time delays.
  • Out-of-band y bypass de filtros vía XML encoding.
