ACT_03 - Parcial 1

Firewall con IPTables

> Linux > Netfilter > Firewall

# Introduccion

IPTables es una herramienta de firewall integrada en el kernel de Linux que permite filtrar paquetes de red, realizar NAT y manipular el trafico. Es fundamental para la seguridad de servidores Linux.

# Estructura de IPTables

# Tablas principales:
- filter: Filtrado de paquetes (INPUT, OUTPUT, FORWARD)
- nat: Traduccion de direcciones (PREROUTING, POSTROUTING)
- mangle: Modificacion de paquetes
- raw: Excepciones al connection tracking

# Comandos Basicos

# Ver reglas actuales
$ sudo iptables -L -n -v

# Limpiar todas las reglas
$ sudo iptables -F

# Politica por defecto: DROP
$ sudo iptables -P INPUT DROP
$ sudo iptables -P FORWARD DROP
$ sudo iptables -P OUTPUT ACCEPT

# Reglas de Seguridad Implementadas

# Permitir loopback
$ sudo iptables -A INPUT -i lo -j ACCEPT

# Permitir conexiones establecidas
$ sudo iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT

# Permitir SSH (puerto 22)
$ sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT

# Permitir HTTP/HTTPS
$ sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
$ sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT

# Bloquear ping floods
$ sudo iptables -A INPUT -p icmp --icmp-type echo-request -m limit --limit 1/s -j ACCEPT

# Registrar paquetes denegados
$ sudo iptables -A INPUT -j LOG --log-prefix "IPTables-Dropped: "

# Proteccion contra Ataques

# Proteccion contra SYN Flood
$ sudo iptables -A INPUT -p tcp --syn -m limit --limit 1/s --limit-burst 3 -j ACCEPT

# Bloquear paquetes invalidos
$ sudo iptables -A INPUT -m state --state INVALID -j DROP

# Proteccion contra port scanning
$ sudo iptables -A INPUT -p tcp --tcp-flags ALL NONE -j DROP
$ sudo iptables -A INPUT -p tcp --tcp-flags ALL ALL -j DROP

# Guardar Reglas

# Guardar reglas (Debian/Ubuntu)
$ sudo iptables-save > /etc/iptables/rules.v4

# Restaurar reglas
$ sudo iptables-restore < /etc/iptables/rules.v4

# Para persistencia, instalar iptables-persistent
$ sudo apt install iptables-persistent
