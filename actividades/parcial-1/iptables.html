ACT_03 - Parcial 1

Interpretacion y traduccion de politicas en iptables

Articulo sobre el filtrado de trafico en Linux mediante iptables, interpretando reglas y su impacto en seguridad perimetral.

# 1) Introduccion

iptables es la herramienta clasica en Linux para administrar reglas de filtrado de paquetes a nivel kernel (Netfilter). Su uso es clave para aplicar politicas de seguridad como "denegar por defecto", permitir servicios especificos y controlar conexiones segun protocolo, puertos y estado de la sesion.

# 2) Desarrollo

Conceptos base

  • Cadenas: INPUT (hacia el host), OUTPUT (desde el host), FORWARD (transito entre redes).
  • Acciones: ACCEPT (permitir), DROP (bloquear silencioso), REJECT (bloquear con respuesta).
  • Stateful firewall: filtrado por estado (NEW, ESTABLISHED, RELATED).

Anatomia de una regla

Parte Ejemplo Significado
Cadena -A INPUT Aplica a trafico que entra al host.
Protocolo -p tcp Filtra solo TCP.
Puerto destino --dport 22 Servicio destino (ej. SSH).
Estado -m state --state NEW Conexiones nuevas.
Accion -j ACCEPT Permite el trafico.

Ejemplos (interpretacion / traduccion)

  • Permitir conexiones establecidas:
    iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
    Traduccion: "Deja pasar respuestas y trafico relacionado a sesiones ya abiertas."
  • Bloquear un puerto especifico (ej. Telnet 23):
    iptables -A INPUT -p tcp --dport 23 -j DROP
    Traduccion: "Bloquea conexiones entrantes al servicio Telnet."
  • Permitir SSH solo desde una IP:
    iptables -A INPUT -p tcp -s 192.168.1.10 --dport 22 -m state --state NEW -j ACCEPT
    Traduccion: "Solo el host 192.168.1.10 puede iniciar SSH al servidor."

# 3) Material

Flujo de decision (vista rapida)

Tipo de trafico Cadena tipica Ejemplo
Internet → Servidor INPUT Acceso HTTP/HTTPS al host
Servidor → Internet OUTPUT Actualizaciones / DNS / descargas
LAN → WAN (router/firewall) FORWARD Clientes navegando

Nota: En firewall perimetral, la cadena FORWARD suele ser la mas utilizada para controlar trafico entre redes.

# 4) Reflexion

Una politica bien disenada en iptables reduce superficie de ataque aplicando "denegar por defecto" y habilitando unicamente servicios necesarios. El filtrado por estado ayuda a permitir respuestas legitimas sin abrir el firewall de mas. Errores comunes: abrir puertos innecesarios, no limitar por IP origen y no registrar (logging) eventos relevantes.
