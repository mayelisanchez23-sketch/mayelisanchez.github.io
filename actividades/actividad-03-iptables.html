Actividad 03 · Interpretación y traducción de políticas en iptables

Volver a Parcial 1

Artículo sobre el filtrado de tráfico en Linux mediante iptables, interpretando reglas y su impacto en seguridad perimetral.

1) Introducción

iptables es la herramienta clásica en Linux para administrar reglas de filtrado de paquetes a nivel kernel (Netfilter). Su uso es clave para aplicar políticas de seguridad como “denegar por defecto”, permitir servicios específicos y controlar conexiones según protocolo, puertos y estado de la sesión.

2) Desarrollo

Conceptos base
  • Cadenas: INPUT (hacia el host), OUTPUT (desde el host), FORWARD (tránsito entre redes).
  • Acciones: ACCEPT (permitir), DROP (bloquear silencioso), REJECT (bloquear con respuesta).
  • Stateful firewall: filtrado por estado (NEW, ESTABLISHED, RELATED).
Anatomía de una regla
Parte Ejemplo Significado
Cadena -A INPUT Aplica a tráfico que entra al host.
Protocolo -p tcp Filtra solo TCP.
Puerto destino --dport 22 Servicio destino (ej. SSH).
Estado -m state --state NEW Conexiones nuevas.
Acción -j ACCEPT Permite el tráfico.
Ejemplos (interpretación / traducción)
  • Permitir conexiones establecidas:
    iptables -A INPUT -m state --state ESTABLISHED,RELATED -j ACCEPT
    Traducción: “Deja pasar respuestas y tráfico relacionado a sesiones ya abiertas.”
  • Bloquear un puerto específico (ej. Telnet 23):
    iptables -A INPUT -p tcp --dport 23 -j DROP
    Traducción: “Bloquea conexiones entrantes al servicio Telnet.”
  • Permitir SSH solo desde una IP:
    iptables -A INPUT -p tcp -s 192.168.1.10 --dport 22 -m state --state NEW -j ACCEPT
    Traducción: “Solo el host 192.168.1.10 puede iniciar SSH al servidor.”

3) Material

Flujo de decisión (vista rápida)
Tipo de tráfico Cadena típica Ejemplo
Internet → Servidor INPUT Acceso HTTP/HTTPS al host
Servidor → Internet OUTPUT Actualizaciones / DNS / descargas
LAN → WAN (router/firewall) FORWARD Clientes navegando

Nota: En firewall perimetral, la cadena FORWARD suele ser la más utilizada para controlar tráfico entre redes.

4) Reflexión

Una política bien diseñada en iptables reduce superficie de ataque aplicando “denegar por defecto” y habilitando únicamente servicios necesarios. El filtrado por estado ayuda a permitir respuestas legítimas sin abrir el firewall de más. Errores comunes: abrir puertos innecesarios, no limitar por IP origen y no registrar (logging) eventos relevantes.
